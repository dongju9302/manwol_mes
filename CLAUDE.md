@AGENTS.md

## 필수 작업 순서 (반드시 지킬 것)
1. AGENTS.md와 CLAUDE.md를 읽고 프로젝트 맥락 파악
2. 수행할 작업 목록을 한국어로 나열 후 사용자 승인 대기
3. 승인 후 작업 시작 전 PROMPTS.md에 날짜와 작업 내용 먼저 기록
4. 수정할 파일 전체 내용 확인
5. 한 번에 하나의 파일만 수정
6. 작업 완료 후 체크리스트로 검증

PROMPTS.md 기록은 선택이 아닌 필수이며
작업 시작 전에 반드시 먼저 기록할 것
기록 없이 작업을 진행하면 안 됨

## 작업 규칙
- 수정 후 어떤 파일의 어떤 내용을 바꿨는지 한국어로 설명할 것
- 불필요한 설명, 반복적인 문장은 생략하고 핵심만 간결하게 답할 것
- TypeScript 타입을 항상 명시할 것
- 에러 발생시 원인과 해결방법을 한국어로 설명할 것
- 모든 코드에 한국어 주석을 빠짐없이 달 것
- 함수, 변수, 컴포넌트 모두 역할을 설명하는 주석 포함할 것

## 코딩 규칙
- Next.js 공식 방식을 우선 사용할 것
- 모든 코드에 TypeScript 타입을 명시할 것
- 주석은 항상 별도 줄에 작성할 것
- 입력값은 항상 유효성 검증 후 처리할 것
- 보안을 고려한 방식으로 구현할 것
- 민감한 정보는 환경변수로 관리할 것

## 코딩 규칙 검토
- 작업 완료 후 새로운 패턴이나 방식이 사용된 경우 기존 코딩 규칙에 추가가 필요한지 검토할 것
- 더 나은 방식이 발견되면 기존 규칙 개선을 제안할 것
- 프로젝트 전체 일관성을 유지하는 방향으로 규칙을 발전시킬 것

## PROMPTS.md 기록 범위 (필수)

다음 모든 활동을 PROMPTS.md에 기록한다.

### 기록 대상
- 코드/파일 수정 (어떤 파일, 어떤 라인, 왜)
- 환경 설정 변경 (Docker, EC2, 환경변수, 보안그룹)
- 디버깅 작업 (증상, 진단 과정, 원인, 해결)
- 새로운 명령어 학습/실행 (SSH, Docker, scp, buildx 등)
- 의사결정 (옵션 비교 후 선택한 이유)
- 외부 서비스 작업 (GitHub, Docker Hub, AWS 콘솔 등)

### 기록 형식
각 작업은 다음 항목을 포함한다:
- 제목: 날짜 시간 + 작업명 (예: 2026-05-12 14:30 - EC2 배포)
- 목표: 한 줄
- 진행 내용: 단계별 핵심 5줄 이내
- 결과: 성공/실패 + 핵심 산출물
- 학습 포인트: 있다면
- 다음 단계: 있다면

### 기록 시점
- 작업 시작 전: 목표와 계획 먼저 기록
- 작업 완료 직후: 결과와 학습 추가 기록
- 한 세션에 여러 작업: 각각 별도 섹션으로 기록

### 민감 정보 마스킹 (필수)
다음은 절대 기록하지 않는다:
- 비밀번호 평문
- JWT 토큰 전체 값
- bcrypt 해시 전체
- .pem 키 파일 내용
- API 시크릿

다음은 기록해도 무방하다:
- EC2 퍼블릭 IP
- 이메일 주소 (학습용)
- 공개된 계정명 (GitHub, Docker Hub ID)
- 명령어와 옵션
